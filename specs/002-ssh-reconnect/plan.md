# Implementation Plan: SSH Reconnection

**Branch**: `002-ssh-reconnect` | **Date**: 2026-01-10 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/002-ssh-reconnect/spec.md`

## Summary

Implement a feature that helps users quickly understand when an SSH connection drops and reconnect smoothly. The main features are:

1. **Connection status indicator**: Always visible in the terminal header and updates immediately when state changes
2. **Reconnect confirmation dialog**: Shown on disconnect and offers reconnect/cancel choices
3. **Auto-reconnect**: Can be enabled or disabled per connection, with up to 3 automatic retries
4. **ReconnectService**: Separates reconnect logic to keep it easy to test

## Technical Context

**Language/Version**: TypeScript 5.6+
**Primary Dependencies**: Expo ~52.0.0, React Native 0.76.0, Zustand 5.0+, react-native-ssh-sftp
**Storage**: AsyncStorage (connection settings), expo-secure-store (password encryption)
**Testing**: Jest (unit tests)
**Target Platform**: Android (mobile)
**Project Type**: Mobile (Expo Router)
**Performance Goals**: Detect disconnects within 3 seconds and show state within 1 second
**Constraints**: Consider mobile battery usage and background processing limits
**Scale/Scope**: Reconnect a single active connection; simultaneous reconnect for multiple connections is out of scope
